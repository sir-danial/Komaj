# Wireframes — کماج

> wireframe متنی صفحات کلیدی. mobile-first با نت desktop.
> جهت: **RTL** — لوگو و navigation راست، actionها چپ، خواندن از راست به چپ.
> تاریخ: ۲۰۲۶-۰۴-۱۹
>
> **وضعیت پیاده‌سازی:** §۱ (Header) + §۲ (Home — hero/categories/featured) + §۱۰ (primitiveها در `/_styleguide/`) به‌صورت static پیاده شد. بقیه صفحات منتظر مدل‌های backend.

---

## ۰. قراردادها

- `[دکمه]` = دکمه primary (زعفرانی)
- `(دکمه)` = دکمه secondary
- `|...|` = ورودی
- `▸` = accordion / expandable
- `◯` = radio، `☐` = checkbox
- `🖼` = تصویر
- `≡` = همبرگر منو
- `⊗` = close
- mobile breakpoint: < 768px / desktop: ≥ 1024px

### نکات حیاتی
- **stepper عددی** باید برای شیرینی کیلویی از `inputmode="decimal"` استفاده کند

- مجموع قیمت به‌محض تغییر quantity با HTMX به‌روز شود
- دکمه «افزودن به سبد» sticky در موبایل (پایین)

## ۱۰. state های special

### Empty states
- **سبد خالی**: illustration + «سبد شما خالی است» + [شروع خرید]
- **جستجو بدون نتیجه**: «کالایی با این مشخصات پیدا نشد» + پیشنهاد دسته‌بندی‌ها
- **سفارش ندارید**: «هنوز سفارشی ثبت نکرده‌اید» + [مشاهده محصولات]

### Loading states
- skeleton loaders با رنگ `sand` → `soft-gold` pulse
- HTMX swap با اندیکاتور کوچک

### Error states
- بنر بالای صفحه با `pomegranate` subtle
- فرم validation inline با متن error زیر فیلد

## ۱۲. تصمیمات — تایید شده ۲۰۲۶-۰۴-۱۹

- [x] **Guest checkout فعال** — ثبت‌نام اجباری نیست. OTP فقط اگر کاربر خواست حساب بسازد.
- [x] **بدون علاقه‌مندی (wishlist)** — حذف شد. (فاز ۳ اگر درخواست شد)
- [x] **کد تخفیف/کوپن فاز ۱** — بخش «کد تخفیف» در صفحه سبد (دسکتاپ) و checkout (قبل از پرداخت) الزامی است.
- [ ] چت آنلاین/پشتیبانی — فعلاً باز
- [ ] selection شهر در home یا فقط checkout — در checkout کافی است

### پیامدهای تصمیمات روی صفحات

**Guest checkout:**
- صفحه OTP (§۷) در مسیر اختیاری قرار می‌گیرد، نه در ابتدای checkout.
- در checkout مرحله ۱ (آدرس): فرم بدون ورود باز می‌شود. فقط فیلد «شماره تماس» + «ایمیل (اختیاری)» اضافه می‌شود برای ارسال رسید.
- در انتهای checkout پس از پرداخت موفق: بنر کوچک «می‌خواهید حساب بسازید تا سفارش‌هایتان را پیگیری کنید؟ [ساخت حساب با OTP]»

**حذف wishlist:**
- آیکون ♡ از product detail حذف
- گزینه «علاقه‌مندی‌ها» از داشبورد حساب حذف

**کد تخفیف:**
- موقعیت: صفحه سبد (قبل از تسویه) و checkout مرحله ۳ (قبل از پرداخت)
- UI: `|کد تخفیف|  [اعمال]` → در صورت موفق، در خلاصه یک ردیف «تخفیف: - ۵۰٬۰۰۰» با رنگ pistachio
- در صورت کد نامعتبر: پیام inline با pomegranate
